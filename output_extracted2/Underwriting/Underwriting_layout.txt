SOP for Query Module in Finverse
Description:

Query Module provides the feasibility to raise a query at any stage from Lead generation till
closing of the Loan application in Finverse. Based on the user role and to whom the query to
be raised, Finverse has Query Tab to raise the query and response as well.

The following features are covered in this document which would explain the flow of Query
Module to different groups.
I.
Raise a query To
i.
Internal
ii.
Agency
iii.
Customer
II.
Response to the query From
i.
Internal
ii.
Agency
iii.
Customer
III.
Query Status
Process:
I.
Raise a Query

At any User level (for raising or understanding to central team) or module level (for case level
queries), there is provision for the user to raise a
Clarification required
(master
based)/
Document Required
specific to application/
Clarification required and document
required
either to Internal or Agency or customer from Query module in Finverse

The queries are defined based on the module/stage (master driven) however the query can be
raised to the customer only after completing sourcing section.

Query module can also be raised used by Supervisor/Skip Level Managers if they want to provide
any clarification/mitigants with regard to particular case

Fields available in Query Module:
a)
Category
: This field is having drop down values: Underwriting, Collateral Technical, Collateral
Legal, Collateral TSR. User can select respective value from the drop down to specify the
process for which he wants to raise query.
b)
Sub- Category–
Query sub-category details are majorly classified in to 3 heads from masters they are:
i.
Raise Internally: Query will be raised to internal teams of ABHFL i.e., Demog Data Entry
or Financial Data Entry etc.
ii.
Raise to Agency: Query will be raised to external agencies which are vendors of ABHFL
i.e.,  Legal Agency, Technical Agency, FI Agency & PD Agencies.
iii.
Raise to Customer: Query will be raised to the respective customer with regard to their
Loan application or for any other additional documents required by ABHFL.
Sub-category Details
are majorly classified to three heads as per the masters:
i.
Clarification Required
ii.
Document required
iii.
Clarification and Document Required.
/media/Underwriting/images/page22_img2.jpeg
/media/Underwriting/images/page22_img3.jpeg
/media/Underwriting/images/page22_img4.jpeg

c)
Assign to
:Selecting query responder/s
For internal: Initiator of the query will be able to add the responder/s who have worked on the
loan application and has an option to mark CC (optional) to the stake holders of the loan
application.
For Customers: Initiator of the query will be able to add the Responder/s (applicant or co-applicant
or both) which will be auto populated and has an option to mark CC (optional) to the stake holders
of the loan application
For example,
a.
If Sourcing Team is raising a query for ‘Documents required’ and selects the applicant/co-
applicants (To whomever the query related to), then the query will be sent to the
applicant/co-applicant along with CC to respective sales teams (optional) if CC is marked.
b.
If Circle Head also want to get clarification, he also can use this module and raise query.
Similarly Supervisor from other vertical can also use this module to add clarification/mitigates
in particular case.
/media/Underwriting/images/page23_img2.jpeg

d)
Title:
There is “Title” query description field where the initiator can capture the query
description in brief. It is free text and user can briefly describe the query for understanding
of addressee.
e)
Message
: Initiator of the query has the option to type the actual query (Free text) with max up
to 1000 characters.
f)
Link/Upload Document:
Initiator of the query has the option to Link/upload the document and
send the query to respective stakeholders. This will help addressee to understand the subject
document for which query is raised.
/media/Underwriting/images/page24_img2.jpeg
/media/Underwriting/images/page24_img3.jpeg
/media/Underwriting/images/page24_img4.jpeg

II.
Response to the Query
a)
Response to ‘Raise Internally’
1.
Once the internal team receives the query, they can view it on their ‘Query’ screen
and dashboard respectively.
2.
If the query is marked to multiple users/user groups, then any one of the users/user
groups will be able to respond with solution to the query.
3.
The responder will be able to respond to the query by giving inputs and uploading
documents (If required).
4.
After the response is submitted, notifications are triggered to the initiator.
5.
The responder to have an option to ‘Mark the query as Resolved and close the
query or request for Additional Information’.
6.
If the initiator is not satisfied with the query resolution, then the initiator will be
able to request for “additional information” and open the query.
b)
Response to ‘Raise to Customer’:
1.
In case the query is to upload any documents, once the Customer (primary applicant/co-
applicant) receives the query, they will be able to respond through TML (Track my Loan).
2.
After the response is submitted, notifications are triggered to the initiator
3.
If the initiator is satisfied with the query resolution, then the initiator will be able to
mark the query as Resolved and close the query.
III.
Query Status
1.
At any stage of the query, any internal user/user group, who are involved in the
specific query, will be able to check the status, which will be of the following type
a)
Open
b)
Closed
2.
For example, when the query is initiated by the initiator, then the status will be ‘open’
until it reaches to the responder.
3.
If the resolution is marked as resolved, it will be closed.
4.
If the resolution is not accepted by the initiator, then the initiator will un-check the
‘Mark as Resolved’ and the status will be changed to open.
5.
So, the same status will be updated and visible to all the users/user group involved in
the queries.
6.
If query is not closed, the user will not be able to decision the case.
